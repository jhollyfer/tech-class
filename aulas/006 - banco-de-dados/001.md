# SQL para Iniciantes

> Aprenda a conversar com bancos de dados do zero.

---

## O que é SQL?

SQL (**Structured Query Language**) é a linguagem usada para criar, consultar, atualizar e deletar dados em bancos de dados relacionais como **MySQL**, **PostgreSQL**, **SQLite** e **SQL Server**.

Pensa assim: o banco de dados é um armário cheio de gavetas (tabelas), e o SQL é você falando o que quer buscar ou guardar nessas gavetas.

---

## Conceitos Básicos

| Termo                      | O que é                                            |
| -------------------------- | -------------------------------------------------- |
| **Banco de dados**         | Conjunto de tabelas organizadas                    |
| **Tabela**                 | Estrutura com linhas e colunas (tipo uma planilha) |
| **Coluna**                 | Um campo/atributo (ex: nome, idade)                |
| **Linha / Registro**       | Um dado completo (ex: um usuário)                  |
| **Chave Primária (PK)**    | Identificador único de cada linha                  |
| **Chave Estrangeira (FK)** | Coluna que referencia outra tabela                 |

---

## Tipos de Dados Principais

```sql
INT           -- número inteiro
DECIMAL(10,2) -- número com casas decimais (ex: preço)
VARCHAR(100)  -- texto com limite de caracteres
TEXT          -- texto longo
BOOLEAN       -- verdadeiro ou falso
DATE          -- data (2024-01-15)
TIMESTAMP     -- data + hora
```

## Criando Tabelas — `CREATE TABLE`

```sql
CREATE TABLE alunos (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  nome       VARCHAR(100) NOT NULL,
  email      VARCHAR(150) UNIQUE NOT NULL,
  idade      INT,
  ativo      BOOLEAN DEFAULT TRUE,
  criado_em  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Explicando:**

- `PRIMARY KEY` → identificador único
- `AUTO_INCREMENT` → o banco gera o número automaticamente
- `NOT NULL` → campo obrigatório
- `UNIQUE` → não pode repetir
- `DEFAULT` → valor padrão quando não informado

## Inserindo Dados — `INSERT`

```sql
-- Inserindo um registro
INSERT INTO alunos (nome, email, idade)
VALUES ('Maria Silva', 'maria@email.com', 22);

-- Inserindo vários de uma vez
INSERT INTO alunos (nome, email, idade)
VALUES
  ('João Santos', 'joao@email.com', 19),
  ('Ana Lima', 'ana@email.com', 25),
  ('Carlos Souza', 'carlos@email.com', 21);
```

---

## Consultando Dados — `SELECT`

```sql
-- Buscar tudo
SELECT * FROM alunos;

-- Buscar colunas específicas
SELECT nome, email FROM alunos;

-- Buscar com condição
SELECT * FROM alunos WHERE idade >= 20;

-- Buscar com múltiplas condições
SELECT * FROM alunos WHERE idade >= 20 AND ativo = TRUE;

-- Buscar com OR
SELECT * FROM alunos WHERE idade < 18 OR idade > 30;
```

---

## Filtrando com `WHERE`

```sql
-- Igual
SELECT * FROM alunos WHERE nome = 'Maria Silva';

-- Diferente
SELECT * FROM alunos WHERE idade != 22;

-- Maior / Menor
SELECT * FROM alunos WHERE idade > 20;
SELECT * FROM alunos WHERE idade <= 25;

-- Entre dois valores
SELECT * FROM alunos WHERE idade BETWEEN 18 AND 25;

-- Lista de valores
SELECT * FROM alunos WHERE idade IN (18, 19, 20);

-- Busca por texto (LIKE)
SELECT * FROM alunos WHERE nome LIKE 'Ma%';   -- começa com "Ma"
SELECT * FROM alunos WHERE nome LIKE '%Silva'; -- termina com "Silva"
SELECT * FROM alunos WHERE nome LIKE '%an%';  -- contém "an"

-- Nulo
SELECT * FROM alunos WHERE idade IS NULL;
SELECT * FROM alunos WHERE idade IS NOT NULL;
```

---

## Ordenando Resultados — `ORDER BY`

```sql
-- Ordem crescente (padrão)
SELECT * FROM alunos ORDER BY nome;
SELECT * FROM alunos ORDER BY nome ASC;

-- Ordem decrescente
SELECT * FROM alunos ORDER BY idade DESC;

-- Ordenar por múltiplas colunas
SELECT * FROM alunos ORDER BY idade DESC, nome ASC;
```

---

## Limitando Resultados — `LIMIT`

```sql
-- Pegar só os 5 primeiros
SELECT * FROM alunos LIMIT 5;

-- Paginação: pular 10, pegar os próximos 5
SELECT * FROM alunos LIMIT 5 OFFSET 10;
```

---

## Atualizando Dados — `UPDATE`

```sql
-- Atualizar um campo
UPDATE alunos SET idade = 23 WHERE id = 1;

-- Atualizar múltiplos campos
UPDATE alunos
SET nome = 'Maria Souza', email = 'maria.souza@email.com'
WHERE id = 1;
```

> ⚠️ **SEMPRE use WHERE no UPDATE.** Sem WHERE você atualiza TODOS os registros!

---

## Deletando Dados — `DELETE`

```sql
-- Deletar um registro
DELETE FROM alunos WHERE id = 1;

-- Deletar com condição
DELETE FROM alunos WHERE ativo = FALSE;
```

> ⚠️ **SEMPRE use WHERE no DELETE.** Sem WHERE você apaga TUDO!

## Funções de Agregação

```sql
-- Contar registros
SELECT COUNT(*) FROM alunos;
SELECT COUNT(*) FROM alunos WHERE ativo = TRUE;

-- Soma
SELECT SUM(idade) FROM alunos;

-- Média
SELECT AVG(idade) FROM alunos;

-- Menor valor
SELECT MIN(idade) FROM alunos;

-- Maior valor
SELECT MAX(idade) FROM alunos;
```

---

## Agrupando — `GROUP BY`

```sql
-- Contar alunos por status
SELECT ativo, COUNT(*) AS total
FROM alunos
GROUP BY ativo;

-- Média de idade por cidade (se tiver coluna cidade)
SELECT cidade, AVG(idade) AS media_idade
FROM alunos
GROUP BY cidade;
```

### Filtrando grupos com `HAVING`

```sql
-- Cidades com mais de 5 alunos
SELECT cidade, COUNT(*) AS total
FROM alunos
GROUP BY cidade
HAVING COUNT(*) > 5;
```

> `WHERE` filtra linhas. `HAVING` filtra grupos.

## Relacionamento entre Tabelas

### Criando tabelas relacionadas

```sql
CREATE TABLE cursos (
  id    INT PRIMARY KEY AUTO_INCREMENT,
  nome  VARCHAR(100) NOT NULL
);

CREATE TABLE matriculas (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  aluno_id   INT NOT NULL,
  curso_id   INT NOT NULL,
  data       DATE DEFAULT (CURRENT_DATE),

  FOREIGN KEY (aluno_id) REFERENCES alunos(id),
  FOREIGN KEY (curso_id) REFERENCES cursos(id)
);
```

---

## Juntando Tabelas — `JOIN`

### INNER JOIN — só traz quem tem correspondência nos dois lados

```sql
SELECT
  alunos.nome,
  cursos.nome AS curso
FROM matriculas
INNER JOIN alunos ON matriculas.aluno_id = alunos.id
INNER JOIN cursos ON matriculas.curso_id = cursos.id;
```

### LEFT JOIN — traz todos da esquerda, mesmo sem correspondência

```sql
-- Todos os alunos, mesmo os sem matrícula
SELECT
  alunos.nome,
  cursos.nome AS curso
FROM alunos
LEFT JOIN matriculas ON alunos.id = matriculas.aluno_id
LEFT JOIN cursos ON matriculas.curso_id = cursos.id;
```

### Usando apelidos (alias) para deixar mais limpo

```sql
SELECT
  a.nome       AS aluno,
  c.nome       AS curso,
  m.data       AS data_matricula
FROM matriculas m
INNER JOIN alunos a ON m.aluno_id = a.id
INNER JOIN cursos c ON m.curso_id = c.id
ORDER BY m.data DESC;
```

## Modificando Tabelas — `ALTER TABLE`

```sql
-- Adicionar coluna
ALTER TABLE alunos ADD COLUMN telefone VARCHAR(20);

-- Remover coluna
ALTER TABLE alunos DROP COLUMN telefone;

-- Renomear coluna
ALTER TABLE alunos RENAME COLUMN email TO email_pessoal;

-- Mudar tipo de dado
ALTER TABLE alunos MODIFY COLUMN idade SMALLINT;
```

---

## Removendo Tabelas e Banco

```sql
-- Apagar tabela (estrutura + dados)
DROP TABLE alunos;

-- Apagar só os dados, manter estrutura
TRUNCATE TABLE alunos;

-- Apagar banco de dados inteiro
DROP DATABASE escola;
```

## Resumo dos Comandos

| Comando        | O que faz                       |
| -------------- | ------------------------------- |
| `CREATE TABLE` | Cria uma tabela                 |
| `INSERT INTO`  | Insere dados                    |
| `SELECT`       | Busca dados                     |
| `WHERE`        | Filtra resultados               |
| `ORDER BY`     | Ordena resultados               |
| `LIMIT`        | Limita quantidade de resultados |
| `UPDATE`       | Atualiza dados                  |
| `DELETE`       | Remove dados                    |
| `JOIN`         | Junta tabelas                   |
| `GROUP BY`     | Agrupa resultados               |
| `HAVING`       | Filtra grupos                   |
| `ALTER TABLE`  | Modifica estrutura da tabela    |
| `DROP TABLE`   | Remove tabela                   |

---

## Boas Práticas

- ✅ Sempre use `WHERE` em `UPDATE` e `DELETE`
- ✅ Use nomes de tabelas e colunas em **minúsculo com underline** (`nome_completo`)
- ✅ Prefira `NOT NULL` para campos obrigatórios
- ✅ Use índices em colunas usadas em `WHERE` e `JOIN`
- ✅ Faça backup antes de comandos destrutivos
- ❌ Nunca use `SELECT *` em produção — especifique as colunas
