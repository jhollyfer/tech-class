# 🗄️ SQL para Iniciantes

> Cada conceito seguido de um exemplo real. Sem enrolação.

---

## O que é SQL?

SQL é a linguagem que você usa pra **conversar com um banco de dados**.  
Quer buscar dados? SQL. Quer salvar? SQL. Apagar? SQL.

Pensa assim: o banco é um armário cheio de gavetas (tabelas), e o SQL é você dizendo o que quer fazer com essas gavetas.

---

## 1. Tipos de Dados

Antes de criar qualquer tabela, você precisa dizer **que tipo de dado** cada coluna vai guardar.

| Tipo            | O que guarda        | Exemplo                |
| --------------- | ------------------- | ---------------------- |
| `INT`           | Número inteiro      | `25`, `100`            |
| `DECIMAL(10,2)` | Número com vírgula  | `99.90`                |
| `VARCHAR(100)`  | Texto curto         | `'Maria'`              |
| `TEXT`          | Texto longo         | Uma bio, uma descrição |
| `BOOLEAN`       | Verdadeiro ou falso | `TRUE`, `FALSE`        |
| `DATE`          | Data                | `2024-03-15`           |
| `TIMESTAMP`     | Data + hora         | `2024-03-15 10:30:00`  |

---

## 2. Criando Tabela — `CREATE TABLE`

**Conceito:** você define a estrutura — nome da tabela, colunas e tipos.

```sql
CREATE TABLE alunos (
  id        INT PRIMARY KEY AUTO_INCREMENT,
  nome      VARCHAR(100) NOT NULL,
  email     VARCHAR(150) UNIQUE NOT NULL,
  idade     INT,
  ativo     BOOLEAN DEFAULT TRUE,
  criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**O que cada parte faz:**

| Trecho           | Significado                                   |
| ---------------- | --------------------------------------------- |
| `PRIMARY KEY`    | Identificador único de cada linha             |
| `AUTO_INCREMENT` | O banco gera o número automaticamente         |
| `NOT NULL`       | Campo obrigatório — não pode ficar vazio      |
| `UNIQUE`         | Não pode ter dois registros com o mesmo valor |
| `DEFAULT`        | Valor usado quando não informado              |

> 💡 **Dica:** Pensa numa tabela como uma planilha. As colunas são os campos, as linhas são os registros.

---

## 3. Inserindo Dados — `INSERT`

**Conceito:** você passa os valores pra cada coluna que quer preencher.

```sql
-- Um registro
INSERT INTO alunos (nome, email, idade)
VALUES ('Maria Silva', 'maria@email.com', 22);

-- Vários de uma vez
INSERT INTO alunos (nome, email, idade)
VALUES
  ('João Santos', 'joao@email.com', 19),
  ('Ana Lima',    'ana@email.com',  25);
```

> 💡 Não precisa informar `id` e `criado_em` — o banco preenche sozinho com `AUTO_INCREMENT` e `DEFAULT`.

---

## 4. Buscando Dados — `SELECT`

**Conceito:** você diz _o que_ quer buscar e _de onde_.

```sql
-- Tudo da tabela
SELECT * FROM alunos;

-- Só as colunas que você quer
SELECT nome, email FROM alunos;
```

> ⚠️ Em produção, evite `SELECT *` — especifique sempre as colunas. É mais rápido e mais legível.

---

## 5. Filtrando — `WHERE`

**Conceito:** coloca uma condição pra trazer só o que você precisa.

```sql
-- Alunos com mais de 20 anos
SELECT * FROM alunos WHERE idade > 20;

-- Um aluno específico
SELECT * FROM alunos WHERE email = 'maria@email.com';

-- Dois critérios juntos
SELECT * FROM alunos WHERE idade >= 18 AND ativo = TRUE;

-- Um ou outro
SELECT * FROM alunos WHERE idade < 18 OR idade > 60;
```

**Operadores disponíveis:**

| Operador          | Significado         | Exemplo                         |
| ----------------- | ------------------- | ------------------------------- |
| `=`               | Igual               | `WHERE ativo = TRUE`            |
| `!=`              | Diferente           | `WHERE idade != 18`             |
| `>` `<` `>=` `<=` | Comparação          | `WHERE idade >= 20`             |
| `BETWEEN`         | Entre dois valores  | `WHERE idade BETWEEN 18 AND 25` |
| `IN`              | Dentro de uma lista | `WHERE idade IN (18, 19, 20)`   |
| `LIKE`            | Busca por texto     | `WHERE nome LIKE 'Ma%'`         |
| `IS NULL`         | Campo vazio         | `WHERE telefone IS NULL`        |

**Exemplos com LIKE:**

```sql
WHERE nome LIKE 'Ma%'    -- começa com "Ma" → Maria, Marcos...
WHERE nome LIKE '%Silva' -- termina com "Silva"
WHERE nome LIKE '%an%'   -- contém "an" em qualquer posição
```

---

## 6. Ordenando — `ORDER BY`

**Conceito:** define a ordem dos resultados.

```sql
-- Ordem alfabética (A → Z)
SELECT * FROM alunos ORDER BY nome ASC;

-- Mais velhos primeiro
SELECT * FROM alunos ORDER BY idade DESC;

-- Ordenar por mais de um critério
SELECT * FROM alunos ORDER BY idade DESC, nome ASC;
```

---

## 7. Limitando — `LIMIT` e `OFFSET`

**Conceito:** controla quantos registros retornar. Essencial pra paginação.

```sql
-- Só os 5 primeiros
SELECT * FROM alunos LIMIT 5;

-- Página 2 (pula 5, pega os próximos 5)
SELECT * FROM alunos LIMIT 5 OFFSET 5;
```

> 💡 **Fórmula de paginação:** `OFFSET = (pagina - 1) * limite`

---

## 8. Atualizando — `UPDATE`

**Conceito:** modifica dados já existentes.

```sql
-- Atualizar um campo
UPDATE alunos SET idade = 23 WHERE id = 1;

-- Atualizar vários campos
UPDATE alunos
SET nome = 'Maria Souza', email = 'maria.souza@email.com'
WHERE id = 1;
```

> 🚨 **SEMPRE use WHERE no UPDATE.**
> Sem WHERE → você atualiza **todos** os registros da tabela. Sem exceção.

---

## 9. Deletando — `DELETE`

**Conceito:** remove registros da tabela.

```sql
-- Remover um registro
DELETE FROM alunos WHERE id = 1;

-- Remover todos os inativos
DELETE FROM alunos WHERE ativo = FALSE;
```

> 🚨 **SEMPRE use WHERE no DELETE.**
> Sem WHERE → você apaga **tudo**. Não tem Ctrl+Z.

---

## 10. Funções de Agregação

**Conceito:** calculam valores a partir de um conjunto de linhas.

```sql
SELECT COUNT(*)   FROM alunos;                     -- total de alunos
SELECT COUNT(*)   FROM alunos WHERE ativo = TRUE;  -- só ativos
SELECT AVG(idade) FROM alunos;                     -- média de idade
SELECT MAX(idade) FROM alunos;                     -- maior idade
SELECT MIN(idade) FROM alunos;                     -- menor idade
SELECT SUM(idade) FROM alunos;                     -- soma de todas as idades
```

---

## 11. Agrupando — `GROUP BY`

**Conceito:** agrupa linhas com valores iguais pra aplicar funções de agregação.

```sql
-- Contar alunos por status
SELECT ativo, COUNT(*) AS total
FROM alunos
GROUP BY ativo;
```

**Resultado:**

| ativo | total |
| ----- | ----- |
| TRUE  | 18    |
| FALSE | 4     |

```sql
-- Média de idade por cidade
SELECT cidade, AVG(idade) AS media_idade
FROM alunos
GROUP BY cidade
ORDER BY media_idade DESC;
```

### Filtrando grupos com `HAVING`

**Conceito:** `HAVING` filtra _grupos_ — diferente do `WHERE` que filtra _linhas_.

```sql
-- Só cidades com mais de 5 alunos
SELECT cidade, COUNT(*) AS total
FROM alunos
GROUP BY cidade
HAVING COUNT(*) > 5;
```

> 💡 Regra simples:
>
> - `WHERE` → filtra **antes** de agrupar
> - `HAVING` → filtra **depois** de agrupar

## 12. Relacionando Tabelas

**Conceito:** tabelas se conectam através de **chaves estrangeiras (FK)**.

```sql
CREATE TABLE cursos (
  id   INT PRIMARY KEY AUTO_INCREMENT,
  nome VARCHAR(100) NOT NULL
);

CREATE TABLE matriculas (
  id       INT PRIMARY KEY AUTO_INCREMENT,
  aluno_id INT NOT NULL,
  curso_id INT NOT NULL,
  data     DATE DEFAULT (CURRENT_DATE),

  FOREIGN KEY (aluno_id) REFERENCES alunos(id),
  FOREIGN KEY (curso_id) REFERENCES cursos(id)
);
```

> 💡 `aluno_id` na tabela `matriculas` aponta pra `id` na tabela `alunos`. Esse é o vínculo.

---

## 13. Juntando Tabelas — `JOIN`

**Conceito:** combina linhas de duas ou mais tabelas a partir de uma condição.

### INNER JOIN — só traz quem tem par nos dois lados

```sql
SELECT
  a.nome  AS aluno,
  c.nome  AS curso,
  m.data  AS data_matricula
FROM matriculas m
INNER JOIN alunos a ON m.aluno_id = a.id
INNER JOIN cursos c ON m.curso_id = c.id;
```

**Resultado:** só alunos que têm matrícula, e só cursos que têm aluno matriculado.

### LEFT JOIN — traz todos da esquerda, mesmo sem par

```sql
SELECT
  a.nome  AS aluno,
  c.nome  AS curso
FROM alunos a
LEFT JOIN matriculas m ON a.id  = m.aluno_id
LEFT JOIN cursos c     ON m.curso_id = c.id;
```

**Resultado:** todos os alunos aparecem. Quem não tem matrícula aparece com `NULL` no campo curso.

**Resumo visual:**

| JOIN         | Traz                                               |
| ------------ | -------------------------------------------------- |
| `INNER JOIN` | Só quem tem correspondência nos **dois lados**     |
| `LEFT JOIN`  | Tudo da **esquerda** + correspondências da direita |
| `RIGHT JOIN` | Tudo da **direita** + correspondências da esquerda |

---

## 14. Modificando Estrutura — `ALTER TABLE`

**Conceito:** muda a estrutura de uma tabela que já existe.

```sql
-- Adicionar coluna
ALTER TABLE alunos ADD COLUMN telefone VARCHAR(20);

-- Remover coluna
ALTER TABLE alunos DROP COLUMN telefone;

-- Mudar tipo de dado
ALTER TABLE alunos MODIFY COLUMN idade SMALLINT;
```

---

## 15. Removendo Tudo — `DROP` e `TRUNCATE`

```sql
-- Apaga a tabela inteira (estrutura + dados)
DROP TABLE alunos;

-- Apaga só os dados, mantém a estrutura
TRUNCATE TABLE alunos;

-- Apaga o banco todo
DROP DATABASE escola;
```

> 🚨 Sem volta. Faça backup antes.

---

## Boas Práticas

✅ Sempre use `WHERE` no `UPDATE` e no `DELETE`  
✅ Nomeie tabelas e colunas em **minúsculo com underline** (`nome_completo`)  
✅ Use `NOT NULL` em campos obrigatórios  
✅ Prefira especificar colunas no `SELECT` em vez de usar `*`  
✅ Use alias (`AS`) pra deixar o resultado mais legível  
❌ Nunca rode `DELETE` ou `UPDATE` sem testar o `WHERE` antes com um `SELECT`
